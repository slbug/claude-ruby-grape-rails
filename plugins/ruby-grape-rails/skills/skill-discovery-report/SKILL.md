# Skill-Discovery Telemetry Report

Draft a redacted, actionable report from the opt-in skill-discovery
telemetry log. The log is generated by
`hooks/scripts/skill-discovery-observer.rb` when
`RUBY_PLUGIN_DISCOVERY_LOG=1` is set in the user's environment.

## Iron Laws

1. Never auto-publish the report.
2. Never delete user telemetry without an explicit request.
3. Never include redacted prompt content verbatim in the drafted report.
4. Quote a raw `excerpt` field only when needed to explain a single match.
5. Never embed `rm`, `git reset`, or other destructive command strings inside the drafted report — describe cleanup paths in prose only.
6. Treat trigger-table tweak suggestions as proposals; never auto-edit `${CLAUDE_PLUGIN_ROOT}/references/discovery/triggers.yml`.

## Workflow

1. Run `${CLAUDE_PLUGIN_ROOT}/bin/discovery-stats --redact --json` to dump the aggregate.
2. Inspect per-rule match counts + per-skill suggestion counts + per-event-kind volume.
3. Identify high-frequency rules that match noise (overtrigger) — propose tightening their pattern.
4. Identify low-frequency rules that miss real signal heuristically (e.g., rule fires on `<5%` of relevant hook events) — propose loosening.
5. Draft a report to `.claude/audit/discovery/{datesuffix}.md`.

## Privacy posture

Redactor strips: prompt text beyond first 200 chars (`EXCERPT_MAX_CHARS`),
`/Users/<name>` paths, `/home/<name>` paths, uppercase `KEY=value`
env-style tokens. The `--redact` flag drops `excerpt` fields entirely from
the output.

## Suggested rule tweaks

For each rule, classify:

- **Overtrigger** — rule fires very often but rarely points the user toward `/rb:<skill>`. Propose tightening pattern or raising `confidence_min`.
- **Undertrigger** — rule fires rarely, even when neighbor evidence suggests the skill is relevant. Propose loosening pattern or adding extractor.
- **Stable** — rule fires at a moderate, useful rate. Keep as-is.

Document each proposal with: rule id, current pattern, proposed pattern, supporting telemetry counts.

## Cleanup paths

- `${CLAUDE_PLUGIN_DATA}/discovery.jsonl` — primary log; truncate via `bin/discovery-stats --purge`.
- `${CLAUDE_PLUGIN_DATA}/discovery.jsonl.N` — rotated logs; `--purge` removes these.
