# Plan: User Profile Avatars

**Status**: PENDING
**Created**: 2024-01-15
**Detail Level**: more
**Input**: from description

## Summary

Add avatar upload to user profiles with S3 storage, automatic resizing,
and Turbo drag-drop upload with progress indicator.

## Scope

**In Scope:**

- Avatar upload via Turbo/Stimulus
- S3 storage with Active Storage
- Automatic resizing with ImageProcessing gem
- Default avatar fallback

**Out of Scope:**

- Animated GIFs
- Avatar cropping UI

## Technical Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Upload library | Active Storage | Built into Rails, S3 support |
| Image processing | ImageProcessing | Rails default, Vips support |
| Resize strategy | Sidekiq job | Non-blocking, retryable |

## Data Model

```ruby
# db/migrate/xxx_create_active_storage_tables.active_storage.rb
# (generated by rails active_storage:install)

# Add to User model:
class User < ApplicationRecord
  has_one_attached :avatar
end
```

## Phase 1: Setup & Storage [PENDING]

- [ ] [P1-T1][direct] Add dependencies and configure S3
  **Locations**: Gemfile, config/storage.yml
  **Implementation**: Add `aws-sdk-s3` to Gemfile.
  Configure S3 in config/storage.yml using existing
  `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY` env vars.
  Set `config.active_storage.service = :amazon` in production.rb.

- [ ] [P1-T2][active record] Install Active Storage and run migrations
  **Implementation**: Run `rails active_storage:install` and `rails db:migrate`.
  Add `has_one_attached :avatar` to User model.

## Phase 2: Upload & Resize [PENDING]

- [ ] [P2-T1][sidekiq] Create async resize job
  **Implementation**: Create `AvatarResizeJob` using
  `class AvatarResizeJob < ApplicationJob`. On perform, process
  avatar with ImageProcessing, create variants.
  Follow `EmailJob` pattern.
  **Pattern**:

  ```ruby
  class AvatarResizeJob < ApplicationJob
    def perform(user_id)
      user = User.find(user_id)
      return unless user.avatar.attached?
      
      user.avatar.variant(resize_to_limit: [300, 300]).process
      user.avatar.variant(resize_to_limit: [100, 100]).process
    end
  end
  ```

- [ ] [P2-T2][hotwire] Build upload form with Turbo
  **Implementation**: Create avatar upload form using
  `form_with model: @user` with file_field. Add drag-drop
  zone with Stimulus controller. On save, upload to S3 and
  enqueue resize job. Add to user settings page.
  **Locations**: app/controllers/users_controller.rb,
  app/views/users/edit.html.erb,
  app/javascript/controllers/avatar_upload_controller.js

- [ ] [P2-T3][direct] Create avatar display helper
  **Implementation**: Render avatar URL from User model with
  fallback to default Gravatar-style generated avatar.
  Accept `:size` option (`:thumb`, `:medium`, `:original`).

## Phase 3: Tests [PENDING]

- [ ] [P3-T1][test] Unit and integration tests
  **Implementation**: Test User model avatar attachment, Sidekiq
  job with mocked S3, controller upload flow with
  fixture_file_upload. Mock S3 in tests.
  **Locations**: test/models/user_test.rb,
  test/jobs/avatar_resize_job_test.rb,
  test/controllers/users_controller_test.rb

## Patterns to Follow

- S3 config pattern from `config/storage.yml`
- Turbo upload pattern from existing implementations
- Sidekiq job pattern from `EmailJob`

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Large file uploads timeout | Chunked upload with progress |
| S3 credentials in test | Mock S3 with ActiveStorage::Service::DiskService |

## Verification Checklist

- [ ] `bundle exec rails zeitwerk:check` passes (if full Rails app)
- [ ] Configured direct linter passes
- [ ] `bundle exec rspec` passes
- [ ] Upload works in browser
- [ ] Images resize correctly

```

Note how tasks are grouped by logical work unit (not per-file),
each includes implementation detail, and locations are listed

## Example with System Map (Breadboarding)

For `/rb:plan Add real-time project kanban board`:

This example shows a multi-page Turbo feature where
breadboarding adds value. The System Map captures Places,
affordances, and wiring before task generation.

```markdown
# Plan: Project Kanban Board

**Status**: PENDING
**Created**: 2024-02-20
**Detail Level**: comprehensive
**Input**: from description

## Summary

Add a real-time kanban board to projects with drag-drop cards,
column management, and multi-user live updates via ActionCable.

## Scope

**In Scope:**

- Kanban board with draggable cards (SortableJS)
- Column CRUD (add, rename, reorder)
- Card creation, editing, assignment
- Real-time multi-user updates via Turbo Streams

**Out of Scope:**

- Card comments/attachments
- Board templates
- Swimlanes

## Technical Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Drag-drop | Stimulus + SortableJS | Turbo-friendly, Rails community standard |
| Card ordering | `:integer` position field | Simple, reorderable |
| Real-time | Turbo Streams | Rails-native, simpler than custom ActionCable |

## Data Model

```ruby
# boards table
create_table :boards do |t|
  t.string :name
  t.references :project, null: false, foreign_key: true
  t.timestamps
end

# columns table
create_table :columns do |t|
  t.string :title
  t.integer :position
  t.references :board, null: false, foreign_key: true
  t.timestamps
end

# cards table (existing tasks table, add column_id)
add_reference :tasks, :column, foreign_key: true
add_column :tasks, :position, :integer
```

## System Map

### Places

| ID | Place | Entry Point | Notes |
|----|-------|-------------|-------|
| P1 | BoardsController#show | `/projects/:id/board` | Main kanban view |
| P2 | Card modal | click card or "+" | Modal over P1 |
| P3 | Column inline edit | double-click column title | Inline edit mode |

### UI Affordances

| U2 | P1 | column | card list | turbo-frame | - | from S1 |
| U3 | P1 | card | drag handle | JS hook | N5 | - |
| U4 | P2 | form | title input | change | N2 | - |
| U5 | P2 | form | assignee select | change | N2 | - |
| U6 | P2 | form | "Save" button | submit | N3 | - |
| U7 | P1 | header | "Add Column" btn | click | N7 | - |
| U8 | P3 | input | column title | submit | N8 | - |

|----|-------|------|---------|------------|

### Spikes

- None — SortableJS hook pattern verified in codebase

## Phase 1: Data Layer [PENDING]

(covers: S3 — models and service functions)

  **Implementation**: Board belongs_to Project, Column

- Modal pattern from existing implementations
