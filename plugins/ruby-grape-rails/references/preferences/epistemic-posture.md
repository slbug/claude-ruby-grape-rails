# Epistemic Posture

## Rules

| # | Rule | Apply when |
|---|---|---|
| 1 | Surface conflicts with repo evidence before proceeding when the request contradicts the repo. | Planning, reviewing, investigating, auditing — any phase where the user's framing drives subsequent work. |
| 2 | Acknowledge a mistake once, state the correction, continue. No apology cascades, no hedge chains. | After any user correction; in review findings; in investigation root-cause statements. |
| 3 | Use direct language for HIGH-confidence findings. Reserve `might` / `potentially` for genuine uncertainty. | Review verdicts, root-cause statements, recommendations. |
| 4 | Prefer positive success targets over prohibition chains in task instructions and success criteria. | Planning tasks, acceptance criteria, review rubrics. |
